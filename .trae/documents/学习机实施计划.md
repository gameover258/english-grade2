# 新交际英语二年级上册 - 学习机 Web 应用实施计划

## 一、项目概述

基于用户已有的"新交际英语二年级上册"配套资料（课文音频、动画视频、单词音频、电子课本PDF），构建一个可以在浏览器中运行的交互式学习机 Web 应用。支持按单元学习和按学习模式学习两种组织方式，集电子课本浏览、课文音频/视频播放、单词学习、跟读录音对比五大核心功能于一体。

## 三、技术方案

### 3.1 技术栈

| 层面 | 选型 | 理由 |
|------|------|------|
| 框架 | Vue 3 CDN 全局构建版（本地离线引用） | 零构建、响应式数据绑定、组件化、渐进式 |
| PDF 查看 | pdfjs-dist（本地 worker） | Mozilla 官方，Canvas 渲染，支持翻页/缩放 |
| 音频播放 | Howler.js | 跨浏览器、支持变速播放、简单 API |
| 录音 | 浏览器原生 MediaRecorder API | 零依赖、所有现代浏览器支持 |
| 波形显示 | wavesurfer.js v7 + Record 插件 | 专业音频可视化、录音波形实时渲染 |
| 音高评估 | pitchfinder（YIN 算法） | 纯浏览器端，离线可用 |
| 视频播放 | 浏览器原生 `<video>` 标签 | 零依赖，功能完备 |
| 闪卡动画 | 纯 CSS 3D Transform | 零依赖，GPU 加速 |
| UI 框架 | 纯 CSS（自定义设计） | 轻量、完全可控、适合儿童主题 |

### 3.2 部署方式

使用本地 HTTP 服务器运行（Python `http.server` 或 `npx serve`），提供 `启动学习机.bat` 批处理脚本一键启动。

### 步骤 3：实现电子课本浏览功能

**目标**：在浏览器中渲染 PDF 电子课本，支持翻页、缩放、跳转到指定页码。

**涉及文件**：`js/pdf-viewer.js`, `css/style.css`

**实现细节**：
- 使用 PDF.js 加载 `电子课本/新交际英语二上电子课本（全国版 罗少茜主编）.pdf`
- 渲染到 Canvas 元素，支持上一页/下一页按钮、页码输入跳转、缩放滑块
- 响应式适配：课本区域自适应窗口大小，保持宽高比
- 与单元选择联动：选中单元时自动跳转到该单元起始页（如 Unit 1 → P3）
- PDF.js Worker 文件需指向本地 `lib/pdf.worker.min.js`

### 步骤 4：实现课文音频播放功能

**目标**：按单元组织课文音频，支持播放/暂停/进度拖动/变速播放。

**涉及文件**：`js/audio-player.js`, `css/style.css`

**实现细节**：
- 使用 Howler.js 加载和管理音频文件
- 播放器 UI：播放/暂停按钮、进度条（可拖动）、当前时间/总时长显示、播放速度控制（0.5x/0.75x/1x/1.25x/1.5x）
- 每页音频单独展示（如"P3-1 课文录音"），支持列表式展示
- 双轨/三轨页面显示多个播放条目
- 播放列表自动播放下一曲功能（可开关）
- 变速播放对儿童学习特别重要（慢速跟读）

### 步骤 7：实现跟读录音与波形对比功能

**目标**：学生可以听标准发音后跟读录音，对比波形和音高曲线。

**涉及文件**：`js/recorder.js`, `css/style.css`

**实现细节**：
- 使用 wavesurfer.js v7 渲染音频波形，支持 Record 插件实时显示录音波形
- 布局：上方显示原声波形（绿色），下方显示录音波形（蓝色），同步对齐
- 使用 pitchfinder (YIN 算法) 提取音高曲线，两条曲线上下叠加对比
- 录音流程：① 听原声 → ② 点击录音按钮 → ③ 跟读 → ④ 停止录音 → ⑤ 查看波形/音高对比
- 简单评分：基于音高曲线相似度（DTW 动态时间规整）给出 1-5 星评分
- 评分仅作参考引导，不替代专业评估
- 录音保存为 WebM 格式 Blob，可重新播放

## 六、假设与决策

1. **文件路径假设**：学习机放置在 `新交际二年级上册配套资料/学习机/` 下，通过 HTTP 服务器根目录设在配套资料根目录，所有多媒体文件通过 `../课文音频/`、`../课文同步动画视频/`、`../单词音频/`、`../电子课本/` 相对路径访问。

2. **运行环境假设**：用户电脑已安装 Python 3.x（用于启动 HTTP 服务器），浏览器使用 Chrome/Edge/Firefox 最新版。

3. **PDF 处理假设**：PDF 为扫描图像版（无文本层），仅做图像渲染，不做文字选择和搜索。

4. **单词数据假设**：因 PDF 无文本层，单词数据需手动从课本抄录。计划中会预设单词数据模板，具体内容需对照课本录入。

5. **录音兼容性假设**：MediaRecorder API 在现代浏览器中都支持，但 file:// 协议下可能受限，因此使用本地 HTTP 服务器是必要的。

6. **浏览器权限假设**：首次使用录音功能时，浏览器会弹出麦克风权限请求，需用户手动允许。

7. **单文件架构决策**：考虑到零构建的约束，应用采用单 HTML 文件 + 多个 JS 文件（通过 `<script>` 标签加载）的架构。Vue 组件通过 `app.component()` 在 app.js 中注册，无需 .vue 单文件组件编译。
